# Iteration 1 Complaints / UX Feedback

This document tracks UX and design complaints for Iteration 1, to be resolved in current or future phases.

## 1. Generic Branding

- **Complaint:** The platform name 'Community Merits' feels a bit generic. The brand should be renamed to **MARS (Merit Activity Records System)** across the pages and emails to give it a more official, system-like identity.
- **Status:** Resolved (implemented in Iteration 2).

## 2. Outdated Matriculation Numbers

- **Complaint:** The student matric numbers displayed are using the outdated registry format (e.g. `U232...`). They must be upgraded to the new alphanumeric standard (e.g. `BC223014`) to test against realistic production data structures.
- **Status:** Resolved (implemented in Iteration 2).

## 3. Browser-Default Date Input

- **Complaint:** The browser-default HTML date input (`<input type="date">`) is inconsistent between Google Chrome, Safari, and Firefox, resulting in poor usability on desktop screens. It should be replaced with a modern custom popover Calendar component.
- **Status:** Resolved (implemented in Iteration 2).
